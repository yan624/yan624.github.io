Transformer位置信息

引用自放弃幻想，全面拥抱Transformer：自然语言处理三大特征抽取器（CNN/RNN/TF）比较 > NLP 句子中单词之间的相对位置是包含很多信息的，上面提过，RNN 因为结构就是线性序列的，所以天然会将位置信息编码进模型；而CNN的卷积层其实也是保留了位置相对信息的，所以什么也不做问题也不大。但是对于 Transformer 来说，为了能够保留输入句子单词之间的相对位置信息，必须要做点什么。为啥它必须要做点什么呢？因为输入的第一层网络是 Muli-head self attention 层，我们知道，Self attention 会让当前输入单词和句子中任意单词发生关系，然后集成到一个 embedding 向量里，但是当所有信息到了 embedding 后，位置信息并没有被编码进去。所以，Transformer 不像 RNN 或 CNN，必须明确的在输入端将 Positon 信息编码，Transformer 是用位置函数来进行位置编码的，而 Bert 等模型则给每个单词一个 Position embedding。将单词 embedding 和单词对应的 position embedding 加起来形成单词的输入 embedding，类似上文讲的 ConvS2S 的做法。

encoder

从文章《可视化理解Transformer结构》做的总结，此文为另一篇英文文章的翻译版，英文原文地址为 The Illustrated Transformer。此外另一篇文章《BERT大火却不懂Transformer？读这一篇就够了》也做了翻译。 Transformer 不同于 RNN，它的输入是独立计算的，输入序列的某个时间步并不依赖其它的时间步。也就是说它可以并行运算。

Positional Encoding

在第一节中讲到，Transformer 天然地不具有句子的位置属性，所以我们需要使用一种办法让它拥有句子的位置属性。 为解决这个问题，Transformer 为每个输入的词嵌入增加了一个向量。

self-Attention

在 此视频（08:30） 中，x 首先要乘一个权重变成 a。但是实际上是一样的，视频中的 x 指的是 one hot，而 a 指的是 embedding。然后将 x 分别乘上三个不同的 transformation 来得到不同的 q, k, v。 > 注意，这些新创建的向量的维度小于词嵌入向量(embedding vector)。它们（新创建的向量）的维度是 64，而词嵌入和编码器的输入输出向量的维度是 512。它们不必更小，这是一种架构选择，可以使多头注意力(multiheaded attention)计算不变。

  1. 需要从每个编码器的输入向量创建三个向量，即 Query 向量，Key 向量和 Value 向量；
    • 那么，究竟什么是“query”，“key”，“value”向量呢？（以下为自己的猜测）
      • q 代表当前需要计算 score 的向量，此向量查询 key 中的权重，使得查询到自己需要的 score
      • k 代表当前单词的权重
      • v 代表单词的词向量
  2. 计算得分（score 权重）。这里的分数是通过将 query 向量与我们正在评分的单词的 key 向量做点积来得到。所以如果我们计算位置 #1 处的单词的 self-attention，第一个得分就是就是 \(q_1\) 和 \(k_1\) 的点积。第二个得分是 \(q_1\) 和 \(k_2\) 的点积。
  3. 将分数除以8（论文中使用 Key 向量维数的平方根---64。这可以有更稳定的梯度。实际上还可以有其他可能的值，这里使用默认值）
  4. 然后经过一个softmax操作后输出结果。Softmax可以将分数归一化，这样使得结果都是正数并且加起来等于1
  5. 将每个 value 向量乘以 softmax 得分（准备将他们相加）
  6. 对加权值向量求和，这样就产生了在这个位置的self-attention的输出（对于第一个单词）

上述为 self-attention 单个单词的计算步骤，其实它可以并行计算。

multi-headed attention

看英文名感觉“高大上”，其实很简单。就是将上述的 self-attention 做多次，具体做几次你可以自行选择，Transformer 选择了 8 次。这样就产生了 8 个向量，但是我们训练时其实只要一个向量就够了，所以我们将这 8 个向量拼接起来，这样就形成了一个巨长无比的向量。那我们怎么得到我们所需长度的向量呢？很简单，只需要再乘一个权重矩阵就行了。

编码器开始处理输入序列，然后将顶部编码器的输出变换为一组注意力向量 K_{encdec} 和 V_{encdec}，这些将在每个解码器的“encoder-decoder attention” 层使用，这有助于解码器集中注意力在输入序列的合适位置。 解释一下 \(K_{encdec}\) 和 \(V_{encdec}\) 的具体用法，在 encoder 中的 self-attention 会使用到 \(Q_{self-attention}\)、\(K_{self-attention}\)、\(V_{self-attention}\) 三个向量，但是它们实际上是输入值 x 的三份拷贝再乘上各自不同的权重矩阵得来。对于 decoder 的 self-attention 与 encoder 的如出一撤，但是对于 encoder-decoder attention 却有点不一样。\(Q_{encoder-decoder \, attention}\) 还是 x 的拷贝乘上一个权重矩阵，但是 \(K_{encoder-decoder \, attention}\) 和 \(V_{encoder-decoder \, attention}\) 分别是 \(K_{encdec}\) 和 \(V_{encdec}\) 乘上各自的权重矩阵。以上所有向量所乘的权重矩阵均可以由你自己随机初始化产生。博主注：以上 QKV 的用法均分析自 pytorch 官方实现版本的源码。 接着下面的 gif 完成了余下的步骤。Transformer 将之前输出的 I 当做下一个时间步的输入，又走了一遍 decoder。以此循环往复，直到输出一个结束标记 <EOS> 才结束循环。

其中 decoder 中的 self-attention 与 encoder 的 self-attenntion 有所区别，一个就是上上段黑体的内容。另外一个是 decoder 的 self-attention 是 masked multi-head attention，而 encoder 的仅仅是 multi-head attention。详见下面 padding mask/sequence mask 一节。 博主注：这样一来似乎对于 decoder 来说并不能实现并行运算。博主注 2：查找资料后，发现是可以的。即如同 encoder 一样，直接将 target 整个序列输入 decoder 中解码。但是这样一来，在做 self-attention 时，提取某个单词的特征时会关注到之后的单词，所以就用到了 sequence mask，这样一来预测的数据其实也倍增了，具体看下面的例子就懂了。不过在模型是实际工作时，由于没有 target 还是只能一次预测一个。

原本的数据只是“我是鱼”，但是由于避免预测时关注到后面的信息，所以只能将部分掩盖住。这其实相当于一个单词一个单词的预测，传统模型中，在预测玩“我”之后，需要输入“是”再预测“鱼”。而 Transformer 中，直接输入“<SOS>我是”，再预测“鱼”，其实这两种方式一模一样。同时为了做到输入“<SOS>”预测“我”，输入“<SOS>我”预测“是”等操作，所以需要拷贝多份数据。

Generator:The Final Linear and Softmax Layer

如何将其转换为一个单词？这就是最后 Softmax 层和线性层的工作了。 线性层是一个简单的全连接神经网络，它将解码器堆叠(decoder stack)产生的向量映射到一个巨大的向量（词汇表的大小，原来的向量大小是词嵌入的大小）中去，这个向量称为 logits 向量。 Softmax 层将这些分数转化为概率(全部为正数，加起来为 1.0)。选择具有最高概率的单元，并将与其相关的单词作为本时间步的输出。

mask

padding mask 是为了防止填充值 PAD 具有意义。在反向传播时，如果不做 padding mask，框架会对 PAD 求导。但是我们认为其实 PAD 是一个没有意义的填充符，所以最好不要对它求梯度。

sequence mask 是为了遮住来自未来的信息，使得 decoder 在解码时，不会依赖未来的信息。

题外话：对于 Transformer，无论 encoder 还是 decoder 都要做 mask（很多对 Transformer 的总结文章都只提到了 decoder 部分的 Masked Multihead Self-Attention，实际上 encoder 也要做一次）。mask 一共分为两种，Transformer 中 decoder 的 mask 指的是 sequence mask，encoder 中的 mask 指的是 padding mask，对于其他网络模型也是类似的道理。可以查阅参考资料 1，2。

padding mask

以 Transformer 为例，在 encoder 中的每次执行 scaled dot-product 后都要做一次 padding mask。由于我们要让序列的长度相等以便做向量化操作，所以必不可少地需要对输入序列进行截断或补零（即填充 PAD，PAD 不一定非要是 0）操作。所以 padding mask 的主要目的是使得 self-attention 不要过多的关注向量中的 PAD 符号。 在进行了解 padding mask 的做法之前，首先简单地了解一下 Transformer 的大致流程：sequence id -> embedding -> multi-head attention(scaled dot-product -> softmax -> softmax \(\times\) value) -> add & norm -> decoder。而 mask 的具体操作是：在执行 scaled dot-product 后，将序列中补零位置所对应的隐藏状态置为 -INF，使得序列经过 softmax 层时，该对应位置所计算出的概率为 0。（mask 操作貌似在 Transformer 中是可选的。） 总得来说，padding mask 是为了神经网络忽略 PAD 的信息。例如，虽然模型可以学习到 PAD 这个填充符是没有意义的，但是在做 attention 时，PAD 所在的这些位置依然有一些 attention score，而我们要的是 0 score。所以有必要引入一个 mask 矩阵，在做 attention 之前，先把 PAD 所在位置的值置为一个极小值，甚至是无穷小（由于 attention 需要经过 softmax，softmax 需要求 e 的次方，要想 e 的某次方为 0，只能使得值为无穷小才可以，这是一个数学问题）。 以下举一些不是 Transformer 的例子： 1. 对于 Embedding 层来说，符号 PAD 的词向量也是无意义的，所以也可以将它的所在位置的值（即词向量）置为 0，使得它无意义。也就是说我们需要引入一个 mask 矩阵，它的大小与词表一样，所有位置初始化为 1，但是 PAD 所在的位置置为 0。然后每次在使用词向量时，乘上这个 mask。不过这样略微麻烦，所幸的是 Pytorch 为我们提供了实现。只需要简单地填入 padding_idx 即可，值为 PAD 对应的索引，索引取决于你的项目。可以看到这非常简单，但是这只是让 PAD 这个词向量无意义并且不计算它的梯度而已。在经过复杂的计算之后 PAD 所在位置的值依旧会变为非 0，而对于其他层想要做到使得 PAD 无意义就没那么简单了。

2. 对于 LSTM 等的时序特征提取层可以使用 torch.nn.utils.rnn.pack_padded_sequence(input, lengths, batch_first=False, enforce_sorted=True) 函数将 embedding 打包。然后将这个打包后的 embedding 输入 LSTM，之后使用 torch.nn.utils.rnn.pad_packed_sequence(sequence, batch_first=False, padding_value=0.0, total_length=None) 函数，将 LSTM 的输出还原。 以以下代码为例，其中第一行代码的 lengths 必须是 pytorch 的 tensor，它代表着你输入序列的长度，比如你输入 [['你', '好', '.'], ['我', '是', '鱼', '.']]，那么 lengths 就是 torch.Tensor([3, 4])。batch_first 默认是 False，这是因为 LSTM 的输入要求是 (S, N, *)，其中 S 是序列长度，N 是批次大小。batch_first 顾名思义就是输入的序列的第一个维度是否为批次大小，显然为了使得 pytorch 内部兼容，batch_first 默认为 False 是最好的。一般我都懒得对序列进行长度上的排序，所以将 enforce_sorted 设置为 False。 3. 如果有些层不像 LSTM 一样只能按照时间步读取，而是可以批量处理时序数据，那么可以使用 mask 矩阵。目前我就知道一个 Transformer 需要用到 mask 矩阵。你可以通过句子的长度自行创建。 4. 最后在计算 loss 时也需要 padding mask，这可以使用 torch.masked_select(input, mask, out=None) 进行计算。

sequence mask

使得 decoder 无法看见未来的信息，其 attention 只能关注解码单词之前的单词，而不能依赖后面未解码出来的单词。这是因为在解码时，由于传统一次只预测一个单词的工作方式太低下，所以 Transformer 的作者做了改进。一次直接将 target 序列输入 decoder，这是可行的，loss 函数还是像往常一样计算即可。例如：

在传统模型中，在预测玩“我”、“是”之后，需要输入“是”再预测“鱼”。而 Transformer 中，直接输入“<SOS>我是”，再预测“鱼”，其实这两种方式一模一样。同时为了做到输入“<SOS>”预测“我”，输入“<SOS>我”预测“是”等操作，所以需要拷贝多份数据。 说白了这个 sequence mask 矩阵其实就是一个 \(L \times L\) 维的矩阵，L 代表一条语句的长度。其内容是，下三角以及对角线全部为 1，代表需要这部分信息。上三角全部为 0，代表这部分信息被掩盖。

最后总结

刚才（2020.1.23）在知乎上看到一篇文章，感觉对 RNN 和 Transformer 的区别总结的不错，所以记录下。 RNN 通过把已经处理过的所有单词的表征与当前正在处理的单词的表征结合起来以此来捕捉整句话的含义，而 Transformer 的 self-attention 机制则是将所有相关单词的含义通过 weighted sum 来充当我们当前正在处理的单词的表征。 参考文章（“从宏观视角看自注意力机制”一节）：
